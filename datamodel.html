Object Reference

Object System Type Key Fields Purpose
NRS Check-in Mobile App Patient Input score (0–10) patientId submittedAt Patient moves the 0–10 slider and taps "Save NRS score". App POSTs to /api/checkin on the Node/Heroku backend, which writes NRS Score __c to Health Cloud via OAuth JWT bearer flow.
PsOdisk Assessment Mobile App Patient Input domains[10] (0–10 each) patientId submittedAt Patient completes 10-domain flow. Same POST to /api/checkin — backend writes PsOdisk Assessment __c + 10 child PsOdisk Domain Score __c records in a single transaction.
Dose Confirmation Mobile App Patient Input confirmed (bool) patientId confirmedAt Patient taps "I took my dose" on the Home screen. App POSTs to /api/dose → Dose Confirmation __c written to Health Cloud. Missed confirmations tracked automatically after daily cutoff.
Agentforce Chat Mobile App Patient Input message sessionId patientId Floating chat bubble on every screen. Calls Agentforce API directly from the backend proxy — backend holds credentials, app never touches SF auth tokens.
Education Hub View Mobile App App UI rankedContentIds[] (received) contentItems[] (rendered) After check-in POST returns, the backend appends the ranked content JSON from MC Next. The Education Hub re-renders with the personalised card order. In demo mode the local personalise() function in contentLibrary.js fills this role.
Contact Health Cloud Standard Id Name DOB Core patient identity record. Anchor for all clinical and engagement data.
CareProgram Enrollment Health Cloud HC Standard Contact__c CareProgram StartDate Status Links the patient to the zasocitinib programme. Controls which content is eligible.
Treatment Record __c Health Cloud Custom Contact__c Drug__c DoseStrength__c StartDate__c TargetWeeks__c Prescription details for zasocitinib 15mg once daily. Source of truth for dose schedule.
NRS Score __c Health Cloud Custom Contact__c Score__c (0–10) SubmittedAt__c Daily patient-reported itch score. Primary personalisation trigger. Streams to Data Cloud on write.
PsOdisk Assessment __c Health Cloud Custom Contact__c AssessmentDate__c OverallScore__c Container record for the 10-domain daily assessment. Parent of Domain Score records.
PsOdisk Domain Score __c Health Cloud Custom Assessment__c Domain__c (picklist) Score__c (0–10) One child record per domain per assessment (10 per session). Domain picklist: itch, pain, sleep, emotional, body, social, work, etc.
Dose Confirmation __c Health Cloud Custom Contact__c ConfirmedAt__c Missed__c Daily dose tracking. Feeds adherence stream in Data Cloud. Triggers At-Risk alert on 2+ missed.
DLQI Assessment __c Health Cloud Custom Contact__c Score__c (0–30) AssessmentDate__c Dermatology Life Quality Index. Completed by patient every 4–6 weeks. Tracked on Progress screen.
Milestone __c Health Cloud Custom Contact__c MilestoneType__c EarnedDate__c Achievement badges: 21-day streak, DLQI improvement, PASI 75 at Week 12. Shown on Progress screen.
Refill Request __c Health Cloud Custom Contact__c DaysRemaining__c RequestedAt__c Status__c Medication refill workflow. Support screen shows alert when <14 days remaining.
Navigator Alert __c Health Cloud Custom Contact__c AlertType__c Severity__c Status__c AgentSuggestion__c Patient Navigator command centre queue. Created by Data Cloud At-Risk segment. Navigator reviews → one-click approve → Agentforce executes outreach.
Unified Individual Data Cloud DMO · Standard UnifiedRecordId ContactPointEmail SourceObjectId Identity-resolved patient across all Health Cloud sources. Anchor for all Data Cloud processing.
NRS Score Stream DMO Data Cloud DMO · Custom IndividualId Score SubmittedAt WeekOnTherapy Ingested from NRS Score __c via HC streaming connector. Source for NRS Trend Calculated Insight.
Assessment Stream DMO Data Cloud DMO · Custom IndividualId Domain DomainScore AssessmentDate Ingested from PsOdisk Domain Score __c. Source for Engagement Score CI (especially emotional domain).
Adherence Stream DMO Data Cloud DMO · Custom IndividualId ConfirmedAt Missed ConsecutiveMissed Ingested from Dose Confirmation __c. Feeds both Engagement Score CI and adherence rate calculation.
NRS Trend (Calc. Insight) Data Cloud Calculated Insight IndividualId AvgNRS_3d AvgNRS_7d TrendDirection Rolling 3-day and 7-day NRS averages. Qualifies patients into High NRS Segment when AvgNRS_3d ≥ 7.
Engagement Score (Calc. Insight) Data Cloud Calculated Insight IndividualId AdherenceRate MissedDoses_7d EmotionalScore RiskScore Multi-signal engagement score. Combines dose adherence, NRS trend, and PsOdisk emotional domain. Qualifies At-Risk Segment.
High NRS Segment Data Cloud Segment Rule: AvgNRS_3d ≥ 7 Triggers itch management content personalisation via MC Next Personalisation Rule. Real-time evaluation after every NRS submission.
At-Risk Segment Data Cloud Segment Rule: MissedDoses_7d ≥ 2 OR NRS trend up 5d OR EmotionalScore < 3 OR 7d no app activity Triggers creation of Navigator Alert __c in Health Cloud. Routes to Patient Navigator command centre + Agentforce proactive outreach queue.
Content Item MC Next on Core MC Object ContentId Title Category Tag ReadTime TriggerConditions Educational articles, guides, and resources. 15 items in demo library. Each has trigger conditions metadata for personalisation rule matching.
Personalisation Rule MC Next on Core MC Object SegmentId ContentIds[] Priority Context Maps Data Cloud segments to ranked content items. Activated when HC receives a segment membership event. Mirrors the local personalise() function in the current demo.
Content Recommendation MC Next on Core API Response RankedContentIds[] PatientContext GeneratedAt Ranked JSON payload returned by MC Next personalisation API after check-in POST. Replaces the local personalise() function when Salesforce is live.
Patient Support Agent Agentforce Agent Actions: answer_faq check_dose_history escalate_to_nurse summarise_report Powers the in-app Agentforce chat bubble. Handles patient queries 24/7, proactively nudges on missed check-ins, and summarises health reports for HCP visits.
Navigator Agent Agentforce Agent Actions: review_alert approve_outreach draft_message Surfaces in the Patient Navigator command centre dashboard. Processes At-Risk queue, drafts personalised outreach messages for Navigator one-click approval.
Proactive Outreach Action Agentforce Agent Action PatientId Channel MessageTemplate ApprovedBy SentAt Navigator-approved outreach executed by Agentforce. Sends SMS, email, or push based on patient preference after Navigator approval.

Note on MC Next on Core: This architecture uses the Salesforce core REST API surface for personalisation — not the legacy ExactTarget / Interaction Studio API. Content decisions flow through Data Cloud segmentation and are executed via Flow or Apex calling the MC Next personalisation endpoint. The local personalise() function in contentLibrary.js mirrors this logic exactly and will be replaced by the API response when the demo org is connected.

Streaming connectors: NRS Score __c, PsOdisk Domain Score __c, and Dose Confirmation __c are written to Health Cloud on check-in submission. A Salesforce–Data Cloud streaming connector ingests these writes in near-real-time, triggering Calculated Insight re-evaluation and segment membership updates within seconds.
